Types in TypeScript

Types by Inference

Unlike JavaScript the TypeScript language was built to generate the types for the user automatically. Therefore, allowing programmers to assign variables to a particular value without specifying the type.

For Example, in TypeScript assigning a value to a variable would look like this,

let helloWorld = "Hello World";

TypeScript can achieve this by knowing how JavaScript works. This allows the language to infer what type the user is assigning to a variable.

Defining Types

To correct possible errors in inferring types in TypeScript you can tell TypeSript what the type should be. You must describe the object using an interface.

For Example, one can explicitly describe an object's shape using the following:

interface User {
  name: string;
  id: number;
}

One can also create an object with an inferred type using the following:
const user = {
  name: "Bob";
  id: 11;
};

Then you can declare that your JavaScript object conforms to the shape defined by your interface by using the following:

const user: User {
  name: "Bob";
  id: 11;
};

If the object doesn't match the interface you defined then TypeScript will warn you:

interface User {
  name: string;
  id: number;
}

const user: User {
  name: "Bob";

Type '{ username: string; id: number; }' is not assignable to type 'User'. Object literal may only specify known properties, and 'username' does not exist in type 'User'.

  id: 11;
};

TypeScript supports object-oriented programming. Therefore, you can use an interface declaration with classes:

interface User {
  name: string;
  id: number;
}

class UserAccount {
  name: string;
  id: number;

constructor(name: string, id: number) {
  this.name = name;
  this.id = id;
  }
}

const user: User = new UserAccount("Bob", 1);

Composing Types

TypeScript allows for one to create their own Types. This can be done by using Unions:

type MyBool = true | false;

In the same manner you are allowed to set what literals are allowed to be:
type motorState = "running" | "stopped";

type doorNumbers = 226 | 336 | 112 | 189;

Using typeof

TypeScript allows the use of the "typeof" operator that can be used to learn the type of a variable.

Below are some examples of how you could check different types:

Type Predicate
String typeof s ==== "string"
number typeof n ==== "number"
boolean typeof b ==== "boolean"
undefined typeof undefined ==== "undefined"
function typeof f ==== "function"
array Array.isArray(a)


The following function returns different values depending on whether its passed in a string or an array:

function wrapInArray(obj: string | string[]) {
  if (typeof obj === "string") {
   return [obj];
  }
  return obj;
}

Generics

In TypeScript one can use generics to provide variables to types. An example of this is an array. An array's values can be described by generics such as:

type StringArray = Array<string>;
type NumberArray = Array<number>;
type ObjectWithNameArray = Array<{ name: string }>;

Migrating From JavaScript

Writing a Config File

TypeScript uses a file called tsconfig.json for managing your project's options, such as which files you want to include and what sorts of checking you want to perform.

Here we're specifying a few things to TypeScript:
  1. Read in any files it understands in the src directory (with include).
  2. Accept JavaScript files as inputs (with allowJs).
  3. Emit all of the output files in built (with outDir).
  4. Translate newer JavaScript constructs down to an older version like ECMAScript 5(using target).

{
  "compilerOptions": {
  "outDir": "./built",
  "allowJs": true,
  "target": "es5"
  }
  "include": ["./src/**/*"]
}

At this point, if you try running tsc at the root of your project, you should see output files in the built directory. The layout of the files in built should look identical to the layout of src. You should now have TypeScript working with your project.
